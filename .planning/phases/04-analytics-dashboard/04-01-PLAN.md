    Internal pipeline (implement exactly in this order):
    1. `const rows = await readRows<Record<string,string>>(`responses-${surveyId}.csv`)` — returns [] if no file
    2. If `rows.length === 0` return `null`
    3. Get Likert question IDs: `GPTW_QUESTIONS.filter(q => q.type === 'likert').map(q => q.id)`
       — This must yield exactly 47 IDs (CAM-01..CAM-08, CRE-09..CRE-17, FAI-18..FAI-26, PRI-27..PRI-35, RES-36..RES-46, UNC-47)
    4. `favorableScore(rows, questionId)` helper: filter out empty strings FIRST, then count "4"|"5", divide by valid count, round to integer. Returns 0 if no valid answers.
    5. Compute per-question scores map: `Record<string, number>`
    6. EES = mean of all 47 per-question scores, rounded
    7. GPTW score = `favorableScore(rows, 'UNC-47')`
    8. Dimension scores: for each of 5 dimensions (camaraderie, credibility, fairness, pride, respect), mean of favorable scores for questions in that dimension. Map dimension names to Title Case for display: `{ camaraderie: 'Camaraderie', credibility: 'Credibility', fairness: 'Fairness', pride: 'Pride', respect: 'Respect' }`
    9. Sentiment: compute across ALL Likert answers (all questions × all rows). Count "4"|"5" as positive, "3" as neutral, "1"|"2" as negative. Divide by total valid Likert answers × 100, round. Skip empty strings.
    10. ENPS from UNC-47: `computeENPS(rows)` — promoters=% of "4"|"5", passives=% of "3", detractors=% of "1"|"2", score=promoters-detractors (signed integer)
    11. Strengths: all 47 per-question scores sorted descending, take 10. Label = question `en` text from GPTW_QUESTIONS. Truncate label to 60 chars with '…' if longer.
    12. Opportunities: same list sorted ascending, take 10.
    13. Leaderboard (11 items in this exact order, use these exact DIMENSION_COLORS values from `src/lib/chart-colors.ts`):
        ```
        { label: 'Completion',  value: responseRate,              color: 'hsl(220 70% 55%)' }
        { label: 'Credibility', value: dimensionScore.credibility, color: 'hsl(220 70% 55%)' }
        { label: 'Respect',     value: dimensionScore.respect,     color: 'hsl(255 55% 58%)' }
        { label: 'Fairness',    value: dimensionScore.fairness,    color: 'hsl(175 45% 45%)' }
        { label: 'Pride',       value: dimensionScore.pride,       color: 'hsl(25 75% 55%)'  }
        { label: 'Camaraderie', value: dimensionScore.camaraderie, color: 'hsl(155 45% 45%)' }
        { label: 'Satisfaction',value: gptwScore,                  color: 'hsl(220 70% 55%)' }
        { label: 'ENPS',        value: enps.score,                 color: 'hsl(155 45% 45%)' }
        { label: 'Engagement',  value: eesScore,                   color: 'hsl(255 55% 58%)' }
        { label: 'Innovation',  value: innovationScore,            color: 'hsl(175 45% 45%)' }
        { label: 'Leadership',  value: leadershipScore,            color: 'hsl(25 75% 55%)'  }
        ```
        Innovation = mean of favorableScore for { CRE-11, RES-38, RES-44, PRI-28 }
        Leadership = mean of favorableScore for { CRE-09, CRE-10, CRE-12, CRE-13, CRE-15 }
        (These question groupings are a discretion area — document with a comment in the code)
    14. Response rate: `const totalTokens = await countSurveyTokens(surveyId)`. If 0, responseRate = 0. Else `Math.round(rows.length / totalTokens * 100)`.
    15. Department breakdown: group rows by `r['DEM-ORG'] || 'Unknown'`. For each group, if `groupRows.length < 5`, all dimension scores = null. Otherwise compute dimension scores. Include `anonymityThreshold: 5`.
    16. Assemble and return `DashboardData` object.
